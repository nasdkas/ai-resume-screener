> Version: 2.1 | Date: 2026-05-13
> 基于项目全部源码（后端 Python/FastAPI、前端 React/TypeScript）的完整审阅

---

## 执行摘要

当前产品在"AI 简历智能评分"这一单点功能上已做得较为深入——双阶段评分（门槛检查 + 维度评分）、服务端强制封顶、锚点校准、置信度计算等机制完善。但作为一款完整的招聘筛选工具，它仍停留在"评分器"阶段，缺少将评分结果转化为招聘行动的链路。

**核心差距**：无用户认证、无候选人状态管理（招聘漏斗）、无搜索/筛选、无可编辑能力、无数据导出、无分页——这些问题使得工具难以在日常招聘场景中闭环使用。

**推荐策略**：优先补齐阻塞核心使用场景的基础能力（分页、通知、搜索、导出、编辑），然后逐步向专业招聘工具演进（看板、协作、Dashboard）。避免过早投入高成本的企业级功能（多租户、合规、移动端）。

**近期已完成**：
- Toast 通知系统（A01）：替换全部 `alert()` / `console.error`，引入 Zustand + Tailwind 自建通知组件
- 自定义确认弹窗（A02+A14）：替换全部 `confirm()`，新建 `ConfirmDialog.tsx` 组件，Promise 模式调用
- CORS 安全配置（A03）：`allow_origins` 从 `["*"]` 改为环境变量 `CORS_ORIGINS` 驱动
- LLM 日志脱敏（A04）：`llm_service.py` 中所有 `print()` 改为 `logger.debug()/error()`，引入标准库 `logging`
- 解析失败重试（A06）：新增 `POST /resumes/{id}/retry` 端点；ResumesPage 和 ResumeDetailPage 增加重试按钮
- JD 选择持久化 localStorage（A05）：Zustand `persist` 中间件 + `partialize` 仅持久化 `selectedJdId`
- 关键词重要性加权体系（必须×3.0/优先×2.0/标准×1.0）已清理，`_format_scoring_criteria` / `_get_must_have_criteria` 已简化
- `keywordMatch` 评分修复：移除 50/50 混合逻辑，改为直接以文本匹配比例计算，避免 LLM 主观评分稀释客观匹配分
- 后端分页（A08）：`storage.py` 新增 `LIMIT/OFFSET` 分页查询；`GET /resumes` 和 `GET /jds` 支持 `page`/`page_size` 参数；前端新建 `Pagination.tsx` 分页组件，ResumesPage 接入分页导航
- 简历解析结果编辑（A07）：新增 `PUT /resumes/{id}` 端点；ResumeDetailPage 支持编辑模式，保存后自动触发重新评分

## 当前架构概述

| 层级 | 技术栈 | 核心职责 |
|------|--------|----------|
| 前端 | React 18 + TypeScript + Vite + Tailwind CSS + Zustand + React Router | 4 个页面：上传、JD 管理、简历列表、简历详情 |
| 后端 | Python 3.11+ + FastAPI + Pydantic + SQLite | 18 个 API 端点，5 个服务模块 |
| LLM | OpenAI API / Ollama（本地部署） | 双阶段评分：门槛检查 + 5 维度打分 |
| 文档解析 | PyPDF2 + python-docx + pdfplumber + PyMuPDF | 支持 PDF/DOCX/TXT |

**数据模型**（4 张表）：`resumes`、`jds`、`match_results`、`failed_uploads`

**评分流程**：上传 → 文本提取 → 门槛检查(Pass 1) → 维度评分(Pass 2) → 加权计算 → 置信度评估 → 存储

## 建议埋点指标

以下指标建议在 Phase 1-2 中逐步加入，用于数据驱动决策：

| 指标 | 用途 | 实现方式 |
|------|------|----------|
| 简历上传成功率 | 监控解析质量 | `parseStatus = 'completed' / total` |

| 导出频率 | 验证 B04 价值 | 计数 `/export/csv` 调用 |
| 编辑率 | 验证 A07 价值 | 计数 `/resumes/{id}` PUT 调用 |
| 搜索使用率 | 验证 B03 价值 | 计数带 `?q=` 参数的请求 |
| 评分校准反馈率 | 验证 C10 价值 | 计数偏高/偏低/合理标记 |
